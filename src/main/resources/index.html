Classicmodels API Reference

The Classicmodels API is organized around REST.

The API has predictable resource-oriented URLs, accepts form-encoded request bodies,
returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

You can test the API with it's Swagger GUI at /swagger-ui/index.html.

Endpoint api/countries

GET /api/countries Retrieve all countries.

Endpoint api/customers

GET /api/customers Retrieve all customers.
GET /api/customers/:customerNumber Retrieve one customer.
GET /api/customers/:customerNumber/purchases Retrieve the purchases (order details) of a customer.
GET /api/customers/:customerNumber/purchaseAggregates Retrieve the aggregated purchases for a customer.
GET /api/customers/:customerNumber/balance Retrieve the balance transactions for a customer.
GET /api/customers/:customerNumber/orders Retrieve the orders of a customer.
GET /api/customers/:customerNumber/ordertotals Retrieve the order totals of a customer.
GET /api/customers/:customerNumber/payments Retrieve the payments of a customer.

Endpoint api/employees

GET /api/employees Retrieve all employees.
GET /api/employees/:emplyeeNumber Retrieve one employee.
GET /api/employees/:employeeNumber/employees Retrieve the subordinates of a manager.

Endpoint api/offices

GET /api/offices Retrieve all offices.
GET /api/offices/:officeCode Retrieve an office.
GET /api/offices/:officeCode/employees Retrieve all employees working in an office.
POST /api/offices Create a new office.
PUT /api/offices/:officeCode Update an existing office.
DELETE /api/offices/:officeCode Delete an office.

Endpoint api/orderDetails

GET /api/orderDetails Retrieve all order details.
POST /api/orderDetails Create a new order detail.

Endpoint api/orders

GET /api/orders Retrieve all orders.
GET /api/orders/:orderNumber Retrieve an order. To get the OrderDetails, use the /api/orderDetails Endpoint.
GET /api/orders/:orderNumber/orderDetails Retrieve the order details for an order.
GET /api/orders/:orderNumber/:productCode Retrieve one single order detail, identified by order number and product code.
POST /api/orders Create a new order. The Json object should contain OrderDetails.
PUT /api/orders/:orderNumber Update an existing order. The Json Object should contain OrderDetails.
DELETE /api/orders/:orderNumber Delete an order. The order's details will be deleted, too.

Endpoint api/payments

GET /api/payments Retrieve all payments.
GET /api/payments/:customerNumber/:checkNumber Retrieve one payment.

Endpoint api/productLineNames

GET /api/productLineNames Retrieve the names of all product lines.

Endpoint api/productLines

GET /api/productLines Retrieve all product lines.
GET /api/productLines/:productLine Retrieve one product line.
GET /api/productLines/:productLine/products Retrieve all products for a product line.

Endpoint api/products

GET /api/products Retrieve all products.
GET /api/products/:productCode Retrieve a product.
GET /api/products/:productCode/orderDetails Retrieve the order details for a product.

Endpoint api/productVendors

GET /api/productVendors Retrieve all product vendors.
